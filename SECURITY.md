# Security Policy

## Reporting a Vulnerability

Do not open a public issue for sensitive security reports.

For a real GitHub Enterprise organization, configure one of these:

- GitHub private vulnerability reporting
- A security email alias
- An internal ticket queue

## Baseline Practices

- Require pull request reviews before merging.
- Require passing status checks.
- Protect the default branch.
- Enable Dependabot alerts where available.
- Use least-privilege access for teams and users.
