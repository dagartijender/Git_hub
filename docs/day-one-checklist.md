# Day One Checklist

Use this checklist when creating a new GitHub Enterprise repository.

## Repository

- [ ] Repository exists in the correct organization.
- [ ] Default branch is named `main`.
- [ ] Repository visibility is correct.
- [ ] Description and topics are set.
- [ ] README exists.
- [ ] License decision is documented.

## Collaboration

- [ ] Issue templates are configured.
- [ ] Pull request template is configured.
- [ ] CODEOWNERS is configured.
- [ ] Required reviewers are known.
- [ ] Team access is configured.

## Automation

- [ ] CI workflow runs on pull requests.
- [ ] CI workflow runs on pushes to `main`.
- [ ] Required status checks are selected in branch protection.

## Governance

- [ ] Branch protection is enabled.
- [ ] Direct pushes to `main` are restricted.
- [ ] Force pushes are disabled.
- [ ] Deleting `main` is disabled.
- [ ] Security reporting process is documented.

## Learning Exercises

- [ ] Open a bug issue.
- [ ] Create a feature branch.
- [ ] Add or change a test.
- [ ] Open a pull request.
- [ ] Review the pull request.
- [ ] Merge after checks pass.
